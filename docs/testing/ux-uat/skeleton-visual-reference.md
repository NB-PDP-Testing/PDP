# Skeleton Loaders Visual Reference

This document provides visual examples for UAT testing of skeleton loading states.

## What Are Skeleton Loaders?

Skeleton loaders are animated gray placeholder boxes that appear while content is loading. They:
- Maintain layout during loading (no layout shift)
- Reduce perceived wait time
- Provide visual feedback that content is coming

## How to Test Skeletons

### Method 1: Network Throttling

1. Open browser DevTools (`F12` or `Ctrl+Shift+I`)
2. Go to **Network** tab
3. Find throttling dropdown (usually says "No throttling")
4. Select **Slow 3G** or **Fast 3G**
5. Navigate to the page you're testing
6. You should see skeleton loaders while data loads

### Method 2: Block API Requests

1. Open browser DevTools
2. Go to **Network** tab
3. Right-click on a Convex request
4. Select "Block request URL"
5. Refresh the page
6. Skeletons should remain visible (since data never loads)

### Method 3: Add Artificial Delay

In dev tools console, you can run:
```js
// This won't work in production, but useful for debugging
window.DEBUG_LOADING_DELAY = 5000; // 5 second delay
```

## Pass/Fail Criteria

| Test | Pass | Fail |
|------|------|------|
| Skeletons appear during loading | Gray animated boxes visible | Blank space or spinner only |
| Layout matches real content | No shift when data loads | Content jumps/shifts |
| Animation is visible | Pulsing or shimmer effect | Static gray boxes |
| Skeletons disappear after load | Real content replaces them | Skeletons remain visible |
| Mobile responsive | Skeletons adapt to screen | Broken layout on mobile |
