## 2026-02-25 - US-P3-001 - Backend: findMatchingYouthProfile Query
**Iteration**: 1
**Commit**: b2decbdfd552d486d31048b88ecb7afddac6a3cb
**Status**: Complete

### What was implemented
- Created packages/backend/convex/models/playerMatching.ts
- Public `findMatchingYouthProfile` query for frontend use (manual add, CSV, invite)
- Internal `findMatchingYouthProfileInternal` internalQuery for mutations/actions
- Shared handler `findMatchingYouthProfileHandler` typed as `GenericQueryCtx<DataModel>`
- Algorithm:
  - PRIORITY 0: by_name_dob exact index to HIGH confidence immediately
  - PRIORITY 1: Fetch org enrollments via by_organizationId -> batch-fetch playerIdentities -> filter youth in memory -> fuzzy score
  - Fuzzy scoring: surname similarity >= 0.85 + exact DOB -> HIGH; first name similarity >= 0.85 + DOB -> MEDIUM; surname only >= 0.85 -> LOW
  - Irish alias check (areIrishAliases) using ALIAS_TO_CANONICAL lookup
  - Email boost: exact match boosts confidence one level
  - Opportunistic GAA: foireann match -> boost to HIGH; mismatch -> warningFlag
