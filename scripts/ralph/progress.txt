## Codebase Patterns
**Last Updated**: 2026-02-28 - Iteration 1

### Architecture
- All routes scoped under `/orgs/[orgId]/`
- Better Auth for auth, Convex for backend, shadcn/ui for components
- Two table systems: Better Auth tables use `ctx.runQuery(components.betterAuth.adapter.*)`, app tables use `ctx.db`

### Convex Backend Patterns
- NEVER use `.filter()` ‚Äî always use `.withIndex()`
- All functions need `args` and `returns` validators
- Use `Id<"tableName">` types, not `string`
- Index names include all fields: `by_parent_and_child`
- `v.partial()` does NOT exist in Convex ‚Äî use `v.optional(v.boolean())` individually
- `authComponent.safeGetAuthUser(ctx)` returns a user with `._id` as string (not `.id`)

### Quality Checks
- `npx -w packages/backend convex codegen` ‚Äî must pass before type check
- `npm run check-types` ‚Äî full project type check
- `npx @biomejs/biome check <specific files>` to check only changed files
- Pre-commit hook runs biome at error level ‚Äî warnings allowed, errors block commit

### File Locations (discovered during work)
- Schema: `packages/backend/convex/schema.ts` (~5607 lines)
- Models: `packages/backend/convex/models/<domain>.ts`
- `restrictChildView` added to `coachParentSummaries` table (not voiceNotes)
- Parent-child auth: `packages/backend/convex/models/parentChildAuthorizations.ts`
