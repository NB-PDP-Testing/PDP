## Codebase Patterns
**Last Updated**: 2026-02-27 - Iteration 1

### Architecture
- All routes scoped under `/orgs/[orgId]/`
- Use Better Auth for authentication, Convex for backend
- Organization data isolated via organizationId field

### Convex Backend Patterns
- NEVER use `.filter()` - always use `.withIndex()`
- All functions need `args` and `returns` validators
- Use `Id<"tableName">` types, not `string`
- BetterAuth member schema: `packages/backend/convex/betterAuth/schema.ts` (~line 200)
- BetterAuth tables: use `ctx.runQuery(components.betterAuth.adapter.*)` and `ctx.runMutation(components.betterAuth.adapter.updateOne, { input: { model, where, update } })`
- Custom fields on Better Auth tables need `as any` cast in updates
- `getMembersForAllOrganizations` returns functionalRoles, activeFunctionalRole, pendingRoleRequests, betterAuthRole per membership
- After adding new fields to schema, also update `getMembersForAllOrganizations` returns validator AND the handler type annotations

### Frontend Patterns
- Use shadcn/ui components from `@/components/ui`
- Org role switcher: `apps/web/src/components/org-role-switcher.tsx`
- Membership data (shared): `useMembershipContext()` from `@/providers/membership-provider`
- `MembershipProvider` wraps the app and provides memberships to all children
- Post-login routing: `apps/web/src/app/orgs/current/page.tsx` → `getRedirectRoute()`
- Org dashboard routing: `apps/web/src/app/orgs/[orgId]/page.tsx`

### Quality Checks
- Run in order: `npx -w packages/backend convex codegen` → `npm run check-types` → `npx ultracite fix` → `npm run check`
- Biome: max 4 function parameters (use options object for 5+)
- Biome: switch statements need `default` clause
- Biome: no single-line if-return (use braces)

### File Locations (discovered during work)
- BetterAuth schema: `packages/backend/convex/betterAuth/schema.ts`
- Members model: `packages/backend/convex/models/members.ts`
- Role switcher: `apps/web/src/components/org-role-switcher.tsx`
- Membership provider: `apps/web/src/providers/membership-provider.tsx`
- Admin settings: `apps/web/src/app/orgs/[orgId]/admin/settings/page.tsx` (large file)
- Player settings: `apps/web/src/app/orgs/[orgId]/player/settings/page.tsx`
- Parents settings: `apps/web/src/app/orgs/[orgId]/parents/settings/page.tsx`
- Coach settings: accessed via `CoachSettingsDialog` in `enhanced-user-menu.tsx`, not a page.tsx
