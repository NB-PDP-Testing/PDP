## Codebase Patterns
**Last Updated**: 2026-02-28 - Iteration 1

### Architecture
- All routes scoped under `/orgs/[orgId]/`
- Two table systems: Better Auth tables use adapter; Application tables use ctx.db
- User ID is `user._id`, NOT `user.id`

### Convex Backend Patterns
- NEVER use `.filter()` - always use `.withIndex()`
- All functions need `args` and `returns` validators
- Use `Id<"tableName">` types, not `string`
- Biome linter requires block statements for all if: `if (!x) { return; }` NOT `if (!x) return;`
- internalMutation/internalQuery imports from `../_generated/server` (same as mutation/query)

### Quality Checks
- Run: `npx -w packages/backend convex codegen` â†’ `npm run check-types`
- 248 pre-existing lint errors â€” new files must pass biome with zero errors
- Pre-commit hook runs `npx biome check --diagnostic-level=error` on staged files only

- Player data: `orgPlayerEnrollments` + `playerIdentities` tables
- Admin pages: `apps/web/src/app/orgs/[orgId]/admin/`
- Convex models: `packages/backend/convex/models/`
