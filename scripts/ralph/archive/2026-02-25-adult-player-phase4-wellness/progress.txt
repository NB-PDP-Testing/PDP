## Codebase Patterns
**Last Updated**: 2026-02-25 - Phase 4 Iteration 1

### Architecture
- All routes scoped under `/orgs/[orgId]/`
- Use Better Auth for authentication, Convex for backend
- Organization data isolated via organizationId field

### Convex Backend Patterns
- NEVER use `.filter()` - always use `.withIndex()`
- All functions need `args` and `returns` validators
- Use `Id<"tableName">` types, not `string`
- Index names include all fields: `by_player_and_date`
- `internalQuery` for read-only internal access; `internalMutation` for internal write
- Use explicit branching in mutations instead of `Record<string, unknown>` patch

### Notification Pattern
- Notification types defined in BOTH schema.ts (v.literal) AND models/notifications.ts AND apps/web/src/components/notification-toast.tsx AND notification-bell.tsx
- When adding a new notification type, update all 4 places

### Quality Checks
- Run in order: `npx -w packages/backend convex codegen` → `npm run check-types` → `npx biome check --write <files>`
- Use `npx biome check --diagnostic-level=error` to see only errors (not warnings)
- Biome rule: `if (!x) continue;` must use block form: `if (!x) { continue; }`
- Biome rule: evolving array types (`const arr = []`) need explicit typing
- Biome rule: NO increment operators (`++`) - use `+= 1`

### File Locations (discovered during work)
- Wellness backend: `packages/backend/convex/models/playerHealthChecks.ts`
- Notification types: `packages/backend/convex/schema.ts`, `models/notifications.ts`, `apps/web/src/components/notification-toast.tsx`, `notification-bell.tsx`
- Player portal pages: `apps/web/src/app/orgs/[orgId]/player/`
