## Codebase Patterns
**Last Updated**: 2026-02-25 - Iteration 1

### Architecture
- All routes scoped under `/orgs/[orgId]/`
- Parent portal at `apps/web/src/app/orgs/[orgId]/parents/` is the reference template
- Player portal at `apps/web/src/app/orgs/[orgId]/player/`

### Convex Backend Patterns
- NEVER use `.filter()` - always use `.withIndex()`
- All functions need `args` and `returns` validators
- adultPlayers.ts had pre-existing lint errors (priority++ in claimYouthProfile) - fixed in iteration 1

### Frontend Patterns
- Portal layouts: "use client" with useOrgTheme(), useUXFeatureFlags()
- Sidebar shown when `adminNavStyle === "sidebar"` (PostHog flag `ux_admin_nav_sidebar`)
- BottomNav shown when `useBottomNav === true` (PostHog flag `ux_bottom_nav`)
- BottomNav uses `id: "overview"` for exact-match active detection
- Portal gating pattern: useQuery(hasPlayerDashboard) → useEffect redirect if false

### Quality Checks
- Run in order: `npx -w packages/backend convex codegen` → `npm run check-types` → biome check
- Pre-commit hook runs `npx biome check --diagnostic-level=error` on ALL staged files
- Pre-existing error: `diagnoseSafeGetAuthUser.ts` - TypeScript error, safely ignore
- `npx ultracite fix` may fail with exit code 1 - run `npx biome check` to verify instead

### File Locations
- Player sidebar: `apps/web/src/components/layout/player-sidebar.tsx`
- Adult players backend: `packages/backend/convex/models/adultPlayers.ts`
- BottomNav: `apps/web/src/components/layout/bottom-nav.tsx`
